基于unity引擎的ARPG手游

2016年6月–2016年9月

Situation

自己一直在玩游戏，内心总有一个游戏人心的心态在呼唤，所以决定自己写一个游戏。就仿照热门ARPG手游模式(太极熊猫拉，天龙八部这种，话说ARPG手游不都一个模子么o(╯□╰)o)，边学边做，就我一个人儿，客户端完成，服务器端实现基本功能。

Task

  • 1.客户端：实现包括角色登陆，背包系统，技能系统等UI系统，实现战斗系统。
  • 2.服务器端：实现服务器登陆验证，账号验证及注册功能。

Action

服务器端

photonserver+NhibernateHelper+MySqlfront

  • 1.服务端
    Application类继承ApplicationBase，每有一个connet请求就创建一个ChatPeer，重写OnOperationRequest()接受he护短的request，通过ChatPeer的sendOperationResponse传回请求结果。主要用工厂模式抽象出不同handler的方法降低耦合度。
    再一个，用Nhibernate做.Net类到数据库的映射，model和mapping处理数据库的字段和属性。

    NHibernateHelper主要帮助链接数据库，通过构建的sessionfactory多commit和query

  • 2.客户端
    主要实现一个IPhotonPerrListener做监听，OnOperationRespone接受服务期短的respone，sendRequest给服务器端发送状态。

Result

通过制作游戏，了解基本游戏制作过程，对DX及图形学的应用也有了进一步的了解。游戏能够实现在手机上的流畅使用，唯一的不足就是独立制作，对服务器端实现功能较少，毕竟一个人经历有限，希望以后能将这个ARPG完善好。游戏的名字暂且叫王者弄鸟吧Σ( ° △ °|||)︴
