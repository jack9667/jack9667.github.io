虚拟现实(htc vive)下犯罪现场取证分析

2016年6月–2017年2月

Situation

为辅助公安或情报部门，针对已获得的关于某场案件中的某些犯罪信息，诸如照片，脚印，行为阶段等信息，利用虚拟现实技术展现犯罪现场并尽可能的进行交互。项目一共5个人，可能的进行交互。项目一共5个人，从事的方面包括照片3D重建，脚印可行性分析，罪犯动作还原，虚拟现实场景实现等几个方面。

Task

我主要完成，罪犯动作展示和虚拟现实场景实现两个方面。

Action

  • 1.罪犯动作还原
    读取以xml文档形式的罪犯信息。每个属性中只包含阶段类型和持续时间两个信息。

    这里遇到一个难点如何实现阶段连续性？1.xml文档不能以目的结果为导向，所以不能使用navmesh做自动寻路。2.每个阶段独立连续所以开启线程为单位略显麻烦。
    主要解决办法利用untiy的协程解决(Coroutines),主要是c#利用yield return和IEnumerator实现的迭代器方法(留个坑具体再补)。主要一点就是多个协程可以同时运行，它们会根据各自的启动顺序来更新，而且协程不是多线程（尽管它们看上去是这样的），它们运行在同一线程中，跟普通的脚本一样；寻路问题就把所有可触碰物体的Colider的IsTrigger设置为ture，让人物行走去留脚印，这样根据脚印分析可行性。
    人物的还原就比较简单了，设计一组Animator的controller，根据不同状态改变parameter展示不同行为即可。

  • 2.虚拟现实场景交互
    设备就用的htc vive
    htc封装了一套SteamVR插件。三个比较重要的prefab
    [Camerarig] 有一个manager类去不断检测手柄
    [status] 一些位置信息
    [steamVR] 渲染组件，
    利用别人的轮子VRTK实现基本物体抓取，位移，touchpad等操作。
    RadialMenu解决touchpah盘，Radial上有Onclick接口，可以实现监听事件。
    SteamVR的接口使用代码也比较简单：

    这方面的应用展现个人觉得比较简单，但难点应该在shader和SteamVR硬件优化上面。

Result

结果实现可以在虚拟现实条件下观察罪犯的可能性行为，做进一步分析。
